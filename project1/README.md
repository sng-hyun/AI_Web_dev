# HTML/CSS를 활용한 넷플릭스 스타일 메인 페이지 구현

## 📌 프로젝트 개요
HTML과 CSS만을 사용하여 넷플릭스 스타일의 메인 페이지를 구현한 프로젝트입니다. 시맨틱 태그를 활용한 구조적 완성도와 사용자 경험을 고려한 인터랙티브 요소를 중점적으로 개발했습니다.

---

## 🎯 구현 섹션 상세

### 1. Header (헤더)
- **구성 요소**: 
  - NFLIX 로고 (좌측)
  - 네비게이션 메뉴 (우측): 홈, 시리즈, 영화, 로그인
- **기술 구현**:
  - `<header>` 시맨틱 태그 사용
  - Flexbox를 활용한 로고와 네비게이션 양 끝 배치 (`justify-content: space-between`)
  - 넷플릭스 시그니처 컬러(#e50914) 적용
  - 호버 효과로 링크 색상 변경

### 2. Hero Banner (메인 홍보 영역)
- **구성 요소**:
  - 메인 타이틀: "무제한으로 영화, 시리즈 등을 즐기세요."
  - 설명 문구: "다양한 디바이스에서 시청하세요. 언제든지 해지 가능합니다."
  - CTA 버튼: "지금 시작하기"
- **기술 구현**:
  - `<section>` 태그로 의미적 구분
  - 중앙 정렬 레이아웃 (`text-align: center`, `margin: 0 auto`)
  - 어두운 배경색(#1a1a1a)으로 시각적 구분
  - 버튼 호버 시 색상 변화 효과

### 3. Content Section (콘텐츠 카드 영역)
- **구성 요소**:
  - 섹션 타이틀: "인기 콘텐츠"
  - 4개의 콘텐츠 카드 (오징어 게임, 기묘한 이야기, 더 크라운, 위쳐)
  - 각 카드: 이미지 + 제목 + 설명
- **기술 구현**:
  - Flexbox Grid 레이아웃 (`display: flex`, `flex-wrap: wrap`, `gap: 20px`)
  - 반응형 카드 배치 (`justify-content: center`)
  - 실제 이미지 로딩 (loremflickr API 활용)
  - **이미지 중복 방지**: URL에 고유 파라미터 추가 (`?random=1`, `?random=2`, `?random=3`, `?random=4`)
  - `object-fit: cover`로 이미지 비율 유지
  - 카드 호버 효과:
    - `transform: scale(1.05)` - 카드 확대
    - 테두리 색상 변경 (#e50914)
  - 오버레이 효과:
    - 기본 상태: `opacity: 0` (투명)
    - 호버 시: `opacity: 1` (불투명)
    - 재생 버튼 + 장르 정보 표시
    - 반투명 검정 배경 (`rgba(0, 0, 0, 0.85)`)

### 4. Footer (푸터)
- **구성 요소**:
  - 2개의 링크 목록 (자주 묻는 질문, 고객 센터, 계정, 미디어 센터 등)
  - 저작권 정보
- **기술 구현**:
  - `<footer>` 시맨틱 태그 사용
  - Flexbox로 링크 목록 가로 배치
  - 호버 시 링크 색상 변경 효과

## ✨ 주요 특징 및 AI 협업 포인트

### 1. 시맨틱 태그 활용
- **구현 내용**: `<header>`, `<main>`, `<section>`, `<article>`, `<footer>` 등 시맨틱 태그를 적절히 사용
- **효과**: 
  - 코드 가독성 향상
  - SEO 최적화
  - 접근성(Accessibility) 개선
- **AI 협업**: 초기 구조 설계 시 AI와 함께 각 섹션에 적합한 시맨틱 태그를 선정하고, `<div>` 남발을 지양하는 방향으로 개발

### 2. 의미 있는 클래스명 사용
- **구현 내용**: `hero-banner`, `content-card`, `card-overlay`, `play-button` 등 BEM 스타일에 가까운 명확한 네이밍
- **효과**: 
  - 유지보수성 향상
  - 코드 이해도 증가
- **AI 협업**: `box1`, `div-wrap` 같은 무의미한 이름 대신 역할을 명확히 나타내는 클래스명을 AI와 함께 설계

### 3. Flexbox Grid 레이아웃 최적화
- **구현 내용**: 
  - `display: flex`와 `flex-wrap: wrap`으로 반응형 카드 배치
  - `gap: 20px`로 카드 간 일정한 간격 유지
  - `justify-content: center`로 중앙 정렬
- **효과**: 
  - 화면 크기에 따라 자동으로 카드 배치 조정
  - 일관된 여백 유지
- **AI 협업**: 초기에는 고정 레이아웃이었으나, AI와 협업하여 반응형 Grid 시스템으로 개선

### 4. 호버 효과 및 Transition 활용
- **구현 내용**:
  - 카드 호버 시 `scale(1.05)` 확대 효과
  - 오버레이 `opacity` 전환 (0 → 1)
  - 모든 효과에 `transition: 0.3s ease` 적용
- **효과**: 
  - 부드러운 사용자 경험 제공
  - 인터랙티브한 느낌 강화
- **AI 협업**: 단순 정적 페이지에서 동적 효과를 추가하는 과정에서 AI와 함께 적절한 transition 시간과 효과를 실험

### 5. 랜덤 이미지 중복 문제 해결 ⭐
- **문제 상황**: loremflickr API를 사용할 때 같은 URL을 사용하면 모든 카드에 동일한 이미지가 표시됨
- **해결 방법**: URL 뒤에 고유 파라미터 추가
  ```html
  <img src="https://loremflickr.com/320/480/movie?random=1" alt="오징어 게임">
  <img src="https://loremflickr.com/320/480/movie?random=2" alt="기묘한 이야기">
  <img src="https://loremflickr.com/320/480/movie?random=3" alt="더 크라운">
  <img src="https://loremflickr.com/320/480/movie?random=4" alt="위쳐">
  ```
- **효과**: 각 카드마다 서로 다른 영화 이미지 표시
- **AI 협업**: 이미지 중복 문제를 AI에게 설명하고, 쿼리 파라미터를 활용한 해결책을 함께 도출

### 6. CSS 코드 간결화 및 주석 보강
- **구현 내용**:
  - 공통 `.container` 클래스로 중복 코드 제거
  - CSS 단축 속성 활용 (`margin: 0 0 20px 0` 등)
  - 입문자를 위한 상세한 주석 추가
- **효과**: 
  - 코드 가독성 향상
  - 유지보수 용이
- **AI 협업**: 코드 리뷰 과정에서 AI와 함께 중복 코드를 찾아내고 최적화

### 7. object-fit: cover 활용
- **구현 내용**: 이미지가 카드 영역을 채우되 비율은 유지
- **효과**: 다양한 비율의 이미지도 깔끔하게 표시
- **AI 협업**: 이미지 비율 문제를 AI와 논의하고 `object-fit: cover` 속성 적용

## 💬 프롬프트 로그

### 대화 1: 이미지 추가 및 중복 방지
**사용자 요청:**
> 현재 작성된 콘텐츠 카드 섹션에 다음 기능을 추가해줘:
> 1. 이미지 추가: 각 카드의 '이미지' 영역에 실제 사진이 보이도록 <img> 태그를 넣어줘. 각 카드마다 서로 다른 영화 이미지가 보일 수 있도록, 주소 뒤에 '?random=1', '?random=2'와 같이 서로 다른 파라미터를 붙여서 작성해줘.

**AI 응답:**
> 각 카드에 실제 이미지를 추가하고, URL에 고유 파라미터를 붙여 서로 다른 이미지를 불러오도록 구현하겠습니다.
> 
> HTML 구조:
> ```html
> <img src="https://loremflickr.com/320/480/movie?random=1" alt="오징어 게임" class="card-img">
> ```
> 
> CSS 스타일:
> ```css
> .card-img {
>     width: 100%;
>     height: 100%;
>     object-fit: cover; /* 이미지 비율 유지하며 영역 채우기 */
>     display: block;
> }
> ```

### 대화 2: 호버 효과 및 오버레이 구현
**사용자 요청:**
> 2. 호버(Hover) 효과 추가: 사용자가 카드 위에 마우스를 올리면 카드가 약간 커지도록(transform: scale) 만들어줘. 마우스를 올렸을 때만 '상세 정보(재생 버튼, 장르 등)'가 나타나는 오버레이(Overlay) 효과를 추가해줘.

**AI 응답:**
> 호버 효과와 오버레이를 구현하겠습니다.
> 
> HTML 구조:
> ```html
> <div class="card-overlay">
>     <button class="play-button">▶ 재생</button>
>     <p class="overlay-genre">생존 게임 드라마</p>
> </div>
> ```
> 
> CSS 스타일:
> ```css
> .content-card:hover {
>     transform: scale(1.05); /* 카드 확대 */
> }
> 
> .card-overlay {
>     opacity: 0; /* 기본 상태: 투명 */
>     transition: opacity 0.3s ease;
> }
> 
> .content-card:hover .card-overlay {
>     opacity: 1; /* 호버 시: 불투명 */
> }
> ```

### 대화 3: 코드 품질 검토 및 개선
**사용자 요청:**
> 작성된 HTML과 CSS 코드를 검토해줘. 특히 시맨틱 태그 사용, 클래스명 의미성, 코드 복잡도를 평가하고 개선해줘.

**AI 응답:**
> 코드를 검토한 결과:
> - ✅ 시맨틱 태그 사용: 우수
> - ✅ 클래스명 의미성: 우수
> - ⚠️ 코드 복잡도: 개선 필요
> 
> 개선 사항:
> 1. 공통 `.container` 클래스 추가로 중복 코드 제거
> 2. CSS 단축 속성 활용 (`margin: 0 0 20px 0`)
> 3. 입문자를 위한 주석 보강
> 4. `calc(100vh - 200px)` → `600px`로 변경 (이해하기 쉽게)
