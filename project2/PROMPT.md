# 📝 To-do List 서비스 개발 프로젝트 정리

## 🏗️ 1. 프로젝트 개요 및 UI 설계
* **목표:** 자바스크립트와 Tailwind CSS를 활용한 고기능 To-do List 웹 서비스 구축
* **UI 구성 원칙:** 간격(Spacing), 정렬(Alignment), 계층(Hierarchy), 반응형(Responsive) 준수
* **화면 구조:**
    * **Header:** 앱 제목
    * **Input Area:** 할 일 입력 및 추가 버튼 (Flex 레이아웃)
    * **Todo List:** 할 일 항목이 동적으로 나열되는 영역
    * **Status Area:** 필터 버튼(전체/진행 중/완료) 및 전체 개수 표시

---

## ⚙️ 2. 핵심 로직 및 데이터 구조
* **데이터 구조:** 각 항목을 객체 `{ id, text, completed, dueDate, deletedAt }` 배열로 관리
* **주요 함수:**
    1.  `renderTodos()`: 상태(필터)에 따른 리스트 화면 출력
    2.  `addTodo()`: 빈 값 검증 후 데이터 추가 및 입력창 초기화
    3.  `toggleTodo(id)`: 완료 상태(true/false) 반전
    4.  `deleteTodo(id)`: 삭제 전 `confirm`창 확인 후 휴지통으로 이동
    5.  `updateStatus()`: 실시간 개수 카운트 및 UI 반영

---

## 📖 [공부] 핵심 개념 및 문제 해결
* **📖 자바스크립트가 화면에 반영되지 않을 때 체크리스트**
    * HTML 파일 하단에 `<script src="script.js"></script>` 연결 여부 확인
    * 데이터 변경 후 반드시 화면을 다시 그리는 `render` 함수 호출 여부 확인
    * 브라우저 개발자 도구(F12) 콘솔 창의 에러 메시지 확인
* **📖 모바일 환경에서의 데이터 저장**
    * 모바일 브라우저도 데스크톱과 동일하게 **LocalStorage**라는 독립된 저장소 제공
    * 기기 내부 메모리에 브라우저별로 데이터가 격리되어 저장됨
* **📖 DOM(Document Object Model)의 역할**
    * HTML 설계도를 자바스크립트가 조작할 수 있는 **'객체 트리'** 형태로 변환한 것
    * JS가 HTML 요소를 찾고, 바꾸고, 생성하고, 이벤트를 감시하는 통로 역할을 함

---

## 🚀 3. 고도화된 고급 기능
* **데이터 영구 저장:** `localStorage`와 `JSON.stringify/parse`를 이용한 데이터 유지
* **항목 수정 (Inline Edit):** 텍스트 클릭 시 입력창으로 전환하여 내용 수정
* **일괄 처리:** 선택된 항목만 삭제하거나 전체 항목 초기화 기능
* **드래그 앤 드롭:** 항목의 순서를 자유롭게 변경하는 인터랙션
* **마감일 관리:** 날짜 설정 기능 및 현재 날짜 기준 D-Day 계산 표시
* **반응형 개선:** 모바일 환경에 최적화된 버튼 크기 및 레이아웃 조정

---

## 🗑️ 4. 특수 기능: 휴지통 시스템
* **이동 로직:** 삭제 시 배열에서 즉시 지우지 않고 `trash` 배열로 타임스탬프(`deletedAt`)와 함께 이동
* **복원 및 영구 삭제:** 휴지통 내에서 다시 목록으로 복구하거나 즉시 삭제 가능
* **30일 자동 삭제:** * 삭제 시점부터 시간을 계산하여 **30일이 경과한 데이터는 앱 실행 시 자동 영구 삭제**
    * 사용자 UI에 "N일 후 삭제됨"과 같은 안내 메시지 표시
